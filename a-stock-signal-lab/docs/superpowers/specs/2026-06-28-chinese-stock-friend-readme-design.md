# 面向股友的中文 README 改版设计

## 目标读者

README 首要服务不写代码、但会使用 Hermes、Codex 或其他 Agent 做 A 股研究的股友。读者应在三分钟内明白项目能做什么、怎样提问、输出代表什么，以及它不能替代哪些判断。

## 股友打开后会看到什么

开头不讲代码、目录和系统架构，先用几句大白话说明：焚诀适合在已经买入、准备做 T、想观察新票，或者拿不准要不要继续持有时，帮忙把逻辑、风险和时机梳理清楚。

接下来直接告诉股友需要提供哪些信息，并准备几段能复制粘贴的问法。例如：“我在 10:30 左右以 12.60 元买了这只票，现在有三成仓，原本看好的是六氟化钨涨价逻辑，帮我判断明天应该继续拿、做 T，还是先防守。”

再往后解释焚诀会给出什么：它不会只丢一个模糊的总分，而是说明建议观察、持有、做 T 或回避的理由，列出风险在哪里、什么情况代表原逻辑失效，以及下一次应该在几点重新检查。

安装命令、专业术语、六策略和工程验证统一放到后面。普通股友看到前半部分就能使用，想自己部署或深入研究的人再继续往下看。README 内不出现“改版范围”“根目录文档”“Skill 内文档”这类维护者才关心的说法。

## 信息顺序

1. 用一句话说明“焚诀”是研究与持仓决策辅助工具，不是荐股机器人。
2. 用股友熟悉的场景介绍能力：已买持仓、做 T、新买观察、竞价与盘中时点、公告监管、底层逻辑。
3. 给出可以直接复制给 Agent 的安装文本和提问模板。
4. 解释用户最好提供的代码、买价、大致时间、仓位、持仓逻辑和操作意图；缺失信息时系统应明确拒绝猜测。
5. 解释输出中的动作、理由、风险上限、失效条件、下一检查点和数据置信度。
6. 将安装命令、CLI、六策略、验证和工程文档放在后半部分，供进阶用户查阅。

## 文案原则

- 全文使用简体中文；首次出现的英文工程术语附中文解释，非必要术语直接删除。
- 不宣传“稳定胜率”“必涨”“精准买卖点”，不把历史命中率写成未来承诺。
- 明确区分研究建议与真实成交；涨跌停、停复牌、数据缺失时不假定可成交。
- 竞价高开区间只作为待验证线索，必须结合板块、情绪、个股股性和承接。
- 保留风险声明，但避免满篇免责声明影响阅读。

## 验收标准

1. 根 README 不看代码也能完成安装并复制一条持仓分析提问。
2. 完整 README 至少包含三个真实使用示例：持仓不动或减仓、做 T、新买观察。
3. 明确列出系统需要的最小输入和会拒绝输出的情况。
4. 所有现有相对链接有效，Markdown 标题层级和代码块闭合。
5. 不删除已有 CLI、六策略、回测治理和工程文档入口，只调整其位置和解释方式。

## 验证方式

- 人工检查两份 README 的中文可读性、链接和代码块。
- 搜索旧版夸大胜率表述，确保只以风险提示形式出现。
- 运行现有文档/CI 契约测试，确认 README 改版未破坏工程质量门。
